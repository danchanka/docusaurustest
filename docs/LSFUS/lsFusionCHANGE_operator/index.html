CHANGE operator#

The CHANGE operator creates an action that changes properties.

Syntax

[CHANGE] propertyId(expr1, ..., exprN) <- valueExpr [WHERE whereExpr]

Description

The CHANGEÂ operator creates an action that changes a property when a condition is met. This operator Â can add its own local parameters when specifying the property whose value is to be changed. These parameters correspond to the objects being iterated and are not parameters of the created action.Â 

The condition is defined by the WHERE block. If this block is not specified, it is assumed that the condition is always met.Â 

The keyword CHANGE, which defines an operator, may be omitted.**Â **

Parameters

propertyId

ID of the property whose value is being changed. The property must be created by certain operators so that its value can be changed. You can change the values of properties created using the operators DATA , ABSTRACT, and LOCAL.

expr1, ..., exprN

A list of expressions or typed parameters defining arguments to the property being changed. When using typed parameters, you can both access already declared parameters and declare new local parameters. When using expressions, new local parameters cannot be added.Â The number of expressions in this list must equal to the number of parameters of the property being changed.Â 

valueExpr

The expression to whose value the property value must be changed.

whereExpr

An expression whose value is a condition for the change being created. If not specified, it is considered equal to TRUE.
