EVAL operator#

The EVAL operator: creating an action that executes code in the lsFusion language.

Syntax

EVAL [ACTION] expression [PARAMS paramExpr1, ..., paramExprN]

Description

The EVAL operator is designed to execute program codeÂ while the server is running. The operator takes as its argument a string containing source code. This stringÂ can be a simple string constant or a more complex expression that evaluates to a string.

The code that is passed should consist of a sequence of instructionsÂ (or action operatorsÂ and local property declarations, if the ACTION option is specified). When the action is executed, this code is inserted into a module with a unique name. The module header is generated by the system: it declares a unique name for the module and lists all the modules that are in the project in the REQUIRE instruction. Thus, you can use all elements from other modules in the code passed in the EVAL operator. If the ACTION option is not specified, it is assumed that one of the passed instructions will be the creation of action with the nameÂ runÂ (it is this action that will be executed).

Parameters

expression

An expression whose value is the string to be executed.

ACTION

Keyword.

paramExpr1, ..., paramExprN

A list of expressions whose values will be passed to the action being executed as arguments. The number of expressions must be equal to the number of parameters of the action being executed.
